# 🐱 Miao 翻译器

一个基于Web Speech API的智能语音识别与情绪化语音合成应用。

## ✨ 功能特色

### 🎤 语音识别
- 实时语音转文字
- 支持连续识别
- 中文语音优化
- 可视化音频指示器

### 😊 智能情绪分析
- 基于关键词的情绪识别
- 支持多种情绪类型：兴奋、可爱风、不满、撒娇、悲伤、惊讶
- 实时置信度显示
- 情绪可视化展示

### 🎭 风格化语音合成
- 多种语音选择
- 可调节语速和音调
- 情绪化语音模板
- 根据情绪自动调整语音参数

### 💭 情绪模板系统
- 预设情绪模板
- 一键应用风格化文本
- 猫咪风格的可爱表达

## 🚀 快速开始

### 环境要求
- 现代浏览器（Chrome、Firefox、Safari、Edge）
- 支持Web Speech API
- 麦克风权限

### 安装使用

1. 克隆或下载项目文件
2. 在浏览器中打开 `index.html`
3. 允许麦克风权限
4. 开始使用！

### 基本操作

#### 语音识别
1. 点击"开始录音"按钮
2. 对着麦克风说话
3. 实时查看识别结果
4. 点击"停止录音"结束识别

#### 情绪分析
- 系统会自动分析识别到的文字
- 检测情绪类型和置信度
- 在情绪分析区域显示结果

#### 语音合成
1. 在文本框中输入要朗读的文字
2. 选择合适的语音
3. 调整语速和音调
4. 点击"开始朗读"

#### 情绪模板
- 点击情绪模板卡片
- 自动填充风格化文本
- 体验不同情绪的语音效果

## 🎯 技术实现

### 第一步：搭建基础框架与音频采集
- 使用HTML、JavaScript和CSS构建单页面应用
- 利用Web Speech API的SpeechRecognition接口进行语音识别
- 配置为持续监听模式，实时捕获音频输入
- 页面包含开始/停止按钮、显示翻译结果的区域以及简单的风格选择按钮

### 第二步：设计情绪识别与风格化映射
- **情绪识别**：通过分析音频的频率、振幅和时长等基本特征，将其映射到预设的几种情绪标签上（如兴奋、可爱风、不满、撒娇等）
- **风格库设置**：为每种情绪标签创建多个有趣的语句模板，如"兴奋"时使用"哇，两脚兽，本喵的御膳还没准备好呢？"，"可爱风"使用"肚肚饿饿啦，想吃小鱼干~"

### 第三步：实现语音合成输出
- 使用Web Speech API的SpeechSynthesis接口
- 将风格化文本传入接口，选择合适的语音进行播放
- 根据情绪调整语音参数（语速、音调等）

## 🔧 技术栈

- **前端框架**：原生HTML/CSS/JavaScript
- **语音识别**：Web Speech API (SpeechRecognition)
- **语音合成**：Web Speech API (SpeechSynthesis)
- **情绪分析**：基于关键词匹配的简单算法
- **UI设计**：现代化响应式设计，支持深色模式

- 渐变背景和毛玻璃效果
- 流畅的动画过渡

- 无障碍设计支持
- 深色模式自适应

## 🔥 支持的情绪类型

| 情绪 | 特征 | 示例模板 |
|------|------|----------|
| 兴奋 | 高频词汇，快语速 | "哇，{text}！太棒了喵！" |
| 可爱风 | 软萌表达，高音调 | "喵~{text}呢~" |
| 不满 | 抱怨词汇，低音调 | "哼！{text}，本喵不开心了！" |
| 撒娇 | 亲昵用词，慢语速 | "嘛~{text}嘛~人家想要~" |
| 悲伤 | 消极词汇，低语速 | 自动调整音调和语速 |
| 惊讶 | 疑问词汇，变化音调 | 自动调整语音参数 |

## 🔒 隐私说明

- 所有语音处理均在本地浏览器中进行
- 不会上传或存储任何音频数据
- 需要麦克风权限仅用于语音识别功能

## 🐛 已知问题

1. 某些浏览器可能需要HTTPS环境才能使用语音识别
2. 移动端浏览器的语音API支持可能有限
3. 网络连接不稳定时可能影响语音识别准确性
